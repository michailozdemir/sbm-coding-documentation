Семантика

Семантика играет одну из важных ролей в построении правильной структуры и определения точных тегов HTML документа. Эта дополнительная информация помогает роботам/индексаторам, таким как Google лучше понять какой контент важен, какой является второстепенным, какой используется для навигации и так далее.

Список самых основных семантических тегов: 

		<header>
<nav>
<main>
<section>
<article>
<aside>
<footer>

Данный раздел документации создан с целью унификации и единого использования одинаковых HTML-тэгов, построение документа страниц SimplyBook.me.

Выбор подходящего HTML-тэга для елемента

Всегда стоит использовать корректный тэг для определенного элемента.
Если это это поле ввода, то input, p - текст, h1-6 - заголовки и т.д.

Не стоит плодить div в теле документа. Никакой семантики в этом нет!

Блок/секция

Каждый блок на странице должен быть обернут в <section>. У <section> может и желательно должны быть свои <header> и <footer>.

Внутри <header> могут находиться:
<h2> - заголовок секции (<h1> если это основной/первый блок или секция)
<p> - подзаголовок секции

Внутри <footer> могут находиться:
Кнопки, кастомные ссылки и т.д.

Пример: 

			<section class="section">

  <header class="section__header">
    <h2 class="hero-title">Block title</h2>
    <p class="subtitle">Block subtitle</p>
  </header>

  <footer class="section__footer">
    <a class="btn" href="linktosomewhere">Block button</a>
  </footer>

</section>

Оформление статьи в секции

Оформление статьи в секции лучшего всего выполнять с помощью <article>. У каждой <article> может быть свой заголовок <h3> и описание<p>.

Пример: 

			<section class="section">

  <header class="section__header">
    <h2 class="hero-title">Block title</h2>
    <p class="subtitle">Block subtitle</p>
  </header>

  <article class="article">
    <h3 class="article__title">What got improved in SimplyBook.me?</h3>
    <p class="article__text">Semantic for sure!</p>
  </article>

  <footer class="section__footer">
    <a class="btn" href="linktosomewhere">Block link</a>
  </footer>

</section>

Оформление карточек, списков, айтемов

Списки, карточки лучше всего оформлять с помощью списочных тэгов HTML. <ul> и <li>. Ведь карточки, это тот же список, только стилистически оформленный в другом виде.

Пример без изображения: 

			<ul class="features">
  <li class="features__item item-features">
    <h3 class="item-features__title">Item title</h3>
    <p class="item-features__text">Item text</p>
  </li>

  <li class="features__item item-features">
    <h3 class="item-features__title">Item title</h3>
    <p class="item-features__text">Item text</p>
  </li>

  <li class="features__item item-features">
    <h3 class="item-features__title">Item title</h3>
    <p class="item-features__text">Item text</p>
  </li>

</ul>

Пример c изображением: 

			<ul class="features">
  <li class="features__item item-features">
    <picture class="item-features__picture"><img class="item-features__image" alt="Image alt text"/></picture>
      <h3 class="item-features__title">Item title</h3>
      <p class="item-features__text">Item text</p>
  </li>
</ul>

Категоризация заголовков

<h1> - единый основной заголовок. Должен быть только один на страницу.
<h2> - для блоков, секций, статей.
<h3> - для айтемов в секциях, блоках.

Категоризация текста

<p> - основной текст на странице, в секциях, статьях.
<span> - текст используемый в качестве декоративных элементов.

Изображения

Изображения желательно оборачивать в специальный HTML-тэг <picture>. Само изображение должно иметь data-src атрибут для инициализации lazy-loading (постепенной загрузки) и alt атрибут для текстового описания.

Пример: 

            <picture class="item__picture">
  <img class="item__image" data-src="/img/path" alt="Image title"/>
</picture>
